1. Validation Rules

Pattern matching must cover all variants. If one branch is missing, compilation fails with a diagnostic that names uncovered variants and location.

enums
result := x?
| .red => 0xFF0000
-- Forgot to cover: .blue and .green cases.
Error · MatchNonExhaustive
Match over enum color is non-exhaustive.
Info: Use _ when explicit fallback behavior is intentional.
Warning: Returning mixed kinds across branches causes typed mismatch errors.
