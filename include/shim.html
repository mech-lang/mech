1. Validation Rules

Pattern matching must cover all variants. If one branch is missing, compilation fails with a diagnostic that names the uncovered variants and location.

enums
result := x?
  | .red  => 0xFF0000
-- Forgot to cover: .blue and .green.
Error · MatchNonExhaustive

Match over enum color is non-exhaustive. Add missing variants or a wildcard branch.

Info: Use _ when explicit fallback behavior is intentional.
Warning: Returning mixed kinds across branches causes typed mismatch errors.

2. Tagged Unions

Tagged unions encode alternatives cleanly. They make state transitions visible and help derive predictable control flow.

3. Pattern Matching

Use nested patterns to destructure deeply while preserving type guarantees. Inline tokens like :running and f64 are highlighted for scanability.
